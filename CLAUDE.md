## What this is

`MuteKey` is a macOS menu-bar (status-bar) app that shows a microphone mute/unmute icon. It is an `.accessory` app — no dock icon, no main window, driven entirely from `AppDelegate`.

The app is in an early, skeletal state: `refreshIcon()` currently hard-codes `muted = true` and `statusItemClicked` is a stub. Wiring up real mute state and the global keyboard shortcut (the `KeyboardShortcuts` dependency is already declared but unused) is the expected direction of work.

## Build & run

The build is reproducible through Nix (via [swiftix](https://github.com/stillwind-ai/swiftix), which pins the swift.org toolchain). Two workflows exist:

- **Nix build (CI / reproducible):** `nix build` → produces `./result/bin/MuteKey`.
- **Dev iteration (faster):** enter the dev shell, then use SwiftPM directly:
  - `nix develop` (or rely on `direnv` — `.envrc` runs `use flake` automatically)
  - `swift build` / `swift run MuteKey`

There are no tests and no linter configured.

## Nix specifics (important when touching dependencies)

- Dependencies are vendored for Nix in [nix/](nix/): [nix/workspace-state.json](nix/workspace-state.json) (pinned checkout state) and [nix/default.nix](nix/default.nix) (content hashes), both generated by `swiftpm2nix`. **After changing `Package.swift` dependencies, regenerate these** — the flake exposes the helper as the `swiftpm2nix` package (`nix run .#swiftpm2nix`). A stale `nix/` will make `nix build` fail or use the wrong revision even though `swift build` works.
- [flake.nix](flake.nix) contains a `postConfigure` hook that strips `#Preview` blocks out of the `KeyboardShortcuts` `Recorder.swift` before building. The swift.org toolchain can't expand Xcode's closed-source `#Preview` macro, so this is required for `nix build` to succeed — do not remove it while that dependency is present.
- The dev shell sets `SDKROOT` to the `apple-sdk_15` SDK on Darwin; `Package.swift` targets `.macOS(.v15)` with Swift 6 language mode.

## Code layout

- [Sources/MuteKeyApp.swift](Sources/MuteKeyApp.swift) — `@main` entry point. Manually constructs `NSApplication`, sets `.accessory` activation policy, and installs `AppDelegate`. Deliberately not using SwiftUI's `App`/`@main` scene lifecycle.
- [Sources/AppDelegate.swift](Sources/AppDelegate.swift) — owns the `NSStatusItem`, renders the icon (`mic.slash.fill` / `mic.fill` SF Symbols, red tint when muted), and handles status-item clicks.

Both types are `@MainActor`.
